## 1. Problem Definition

### 1.1 As-Is

An SSL static audit on the v0.1.0 skill bodies (branch `skill-audit/ssl-review`, with self-verification ERRATA) identified three concrete behavior-level gaps:

1. **ralphi orphan reference**: `skills/ralphi/references/criteria-project.md` exists but `skills/ralphi/SKILL.md` Step 3B (Project/Directory Inspection) does not load it. Step 3A maps file types to `criteria-{prd|skill|script|code}.md`, Step 3B has no parallel mapping. Verified: `grep -n "criteria-project" skills/ralphi/SKILL.md` → no hits.
2. **drafti-architect / drafti-feature silent overwrite**: Step 5 / Step 7 HITL asks *"Save to `docs/prd-{slug}.md`?"* but does not check whether the target already exists. Re-running with the same slug overwrites a previously saved PRD without warning. The harnish convention is opt-in writes; silent overwrite is an asymmetric break.
3. **impl orphan reference**: `skills/impl/references/retention-policy.md` documents `purge-assets.sh` flow (asset deletion) but `skills/impl/SKILL.md` Mode Detection table and Step 5 manual triggers do not reference it. The deletion path exists in the harnish_py runtime but is unbound to any skill utterance, leaving destructive semantics off the skill surface.

The audit's ERRATA round also surfaced a *meta-finding* — that the audit's initial proposal of an `ssl:` frontmatter block was non-conventional (would break bilingual `.ko.md` lockstep without precedent). This PRD does **not** adopt the SSL frontmatter; gaps are fixed in body prose, matching the existing "Prohibited / Guardrails / Context Budget" convention used by all 5 skills today.

### 1.2 To-Be

- ralphi Step 3B loads exactly one criteria file (`references/criteria-project.md`), parallel to Step 3A's per-file-type mapping. en + ko in lockstep.
- drafti-architect Step 5 / drafti-feature Step 7 HITL prompt detects a pre-existing `docs/prd-{slug}.md` and offers `overwrite` / `new-slug` / `n` instead of silent overwrite. en + ko in lockstep.
- impl Step 5 manual triggers table includes an entry for retention/purge utterances OR retention-policy.md is explicitly framed as an out-of-band ops doc inside SKILL.md so the orphan status becomes intentional, not accidental. en + ko in lockstep.
- Bats / pytest regression suite verifies the literal phrases / mappings exist, so future edits cannot silently regress them.

### 1.3 Impact

The three gaps were detected by static audit and have not yet caused a user-visible incident. ralphi's Step 3B currently runs without loading any criteria — meaning autonomous fixes in directory scope have **no per-criteria reference**, only built-in heuristics. drafti silent overwrite is a one-keystroke data loss path on PRD re-runs. impl orphan reference means the audit report itself was the first place to surface that purge has destructive semantics. All three are below release blockers but are exactly the class of issue the audit framework was set up to catch — leaving them in a v0.1.0 release after surfacing them undermines the audit pipeline.

## 2. Design Decision

### Alternative A: Body prose fix per existing convention (no frontmatter changes)

- **Approach**: Amend SKILL.md / SKILL.ko.md bodies for ralphi (Step 3B), drafti-architect (Step 5), drafti-feature (Step 7), impl (Step 5 manual triggers + Step 1 Mode Detection note). No frontmatter schema change. No script changes. Bats regression for each literal phrase.
- **Pros**: Symmetric with `docs/prd-impl-midloop-guard.md` (v0.0.3) precedent — that fix used identical pattern (prose addition + bats grep regression). Bilingual lockstep preserved trivially. No new convention to maintain.
- **Cons**: Prose enforcement only — LLM must read and follow. Same enforcement level as the other guards in these skills (which is also prose), so the asymmetry argument cuts the other way.
- **Suitable**: When the gaps are documented-behavior gaps, not runtime behavior gaps. (All three audit findings are documented-behavior gaps.)

### Alternative B: Frontmatter SSL block (the rejected v1 proposal)

- **Approach**: Add `ssl: { scheduling, structural, logical }` to all 5 SKILL.md frontmatters. Mirror to all 5 .ko.md. Update `scripts/test-all.sh` + `scripts/pre-commit.sh` to validate the schema. Bump as a minor (0.2.0) since it introduces a new contract.
- **Pros**: Machine-readable side-effect / anti-trigger / rollback declarations. Audit re-runs become a JSON diff, not prose grep.
- **Cons**: New convention layer; doubles the bilingual surface (every SSL change must be translated); `test-all.sh` deep-tier validation must be extended; sibling repos (honne, galmuri) don't adopt this convention so it diverges. The audit's *content* (anti-triggers, side effects) is already coverable by body prose.
- **Suitable**: When the audit pipeline is going to run frequently and machine-parseable output is the primary product. Not the case for this release.

### Alternative C: Mixed — body prose for two, retire `criteria-project.md` for one

- **Approach**: A for drafti × 2 and impl. For ralphi specifically: delete `criteria-project.md` since it's orphan, and let Step 3B remain criteria-less (which is current behavior).
- **Pros**: One fewer file to maintain.
- **Cons**: criteria-project.md was clearly authored as the directory-scope counterpart to criteria-{prd|skill|script|code}.md — deleting it walks back design intent. The orphan is a *wiring* bug, not a *scope* bug.
- **Suitable**: If we had evidence ralphi's directory scope intentionally avoids criteria. We don't.

### Selection: **A**

- **Rationale**: All three findings are documented-behavior gaps. The v0.0.3 mid-loop guard precedent established prose+bats as the standard pattern for closing this class of gap. SSL frontmatter (B) was the audit's own initial speculation and was retracted in ERRATA after convention check. C contradicts the original `criteria-project.md` design intent.
- **Rejected**: B (over-engineered for documented-behavior gaps; no sibling-repo precedent). C (deletes evidence of design intent rather than wiring it).
- **Validity condition**: If a follow-up release adds a runtime hook that needs to read declared side-effects (e.g., a permission gate), revisit B. Until then, prose+bats is sufficient.

## 6. Test Criteria

Each acceptance check is a single bash one-liner that returns exit 0:

1. ralphi Step 3B mapping (en): `grep -q "criteria-project" skills/ralphi/SKILL.md`
2. ralphi Step 3B mapping (ko): `grep -q "criteria-project" skills/ralphi/SKILL.ko.md`
3. drafti-architect Step 5 overwrite gate (en): `grep -q "overwrite" skills/drafti-architect/SKILL.md && grep -q "new-slug" skills/drafti-architect/SKILL.md`
4. drafti-architect Step 5 overwrite gate (ko): same on `.ko.md`
5. drafti-feature Step 7 overwrite gate (en+ko): same pattern
6. impl retention-policy reference (en+ko): `grep -q "retention-policy" skills/impl/SKILL.md && grep -q "retention-policy" skills/impl/SKILL.ko.md`
7. Version sync deep tier: `bash tests/run.sh` (existing test-all.sh:1037-1053 covers this)
8. Pytest unit tests: `pytest tests/ -q` — no regression in 35+ existing tests
9. Bats e2e: `bats tests/*.bats` — all existing + new bats files pass
10. CHANGELOG entry: `grep -q "## \[0.1.1\]" CHANGELOG.md`

## 7. Guardrails

- **No frontmatter schema additions**. `name / description / version` only — established by existing pre-commit + test-all checks.
- **No script changes**. Wrappers and `harnish_py/` modules are untouched.
- **Bilingual lockstep**: every en edit has a ko mirror in the same commit. Verified by test-all.sh deep-tier.
- **Audit artifacts**: the SSL audit log itself is *not* retained as a release artifact. The PRD captures all actionable findings; the raw audit log is treated as scratch work, like the conversation that produced it. (If future audits warrant retention, establish the convention then.)
- **No new convention layer**. SSL frontmatter (B) is explicitly out of scope for v0.1.1.
- **No dependency changes**. Python 3.14+, no new pip packages.
