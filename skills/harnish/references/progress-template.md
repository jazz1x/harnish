## 갱신 규칙 상세

### 태스크 시작 시

Doing 섹션에 태스크를 추가한다. 필수 필드: 시작 시간, 태스크 컨텍스트.
Todo에서 해당 태스크를 제거한다.

### 3액션마다

Doing의 "현재"와 "마지막 액션"을 갱신한다.
"다음 액션"도 갱신한다.
이 갱신은 세션 중단 시 복원 지점 역할을 한다.

### 태스크 완료 시

1. Doing에서 태스크를 제거한다
2. Done에 추가한다 (결과, 변경 파일, 검증, 소요)
3. Todo에서 다음 태스크를 확인한다
4. 마일스톤이면 체크포인트 보고서를 생성한다

### 에러 발생 시

1. 이슈 로그에 추가한다
2. Doing의 블로커에 기술한다
3. 시도 횟수를 증가시킨다

### 마일스톤 도달 시

1. 요약 통계를 갱신한다
2. 체크포인트 보고서를 생성한다 (별도 출력)
3. 사용자 승인을 요청한다
4. 승인 결과를 기록한다

### 세션 시작 시

1. PROGRESS.md를 읽는다
2. 현재 상태를 파악한다
3. Doing 항목이 있으면 "다음 액션"부터 이어서
4. Doing이 비어있으면 Todo에서 다음 태스크 선택
5. 사용자에게 간략히 보고한다
